type Estado = (Float, Mapa, [PowerMais], [PowerFlame], [Bomba], [Player], [Picture], Int)

parse :: [String] -> Estado -> Estado

Estado == (tmp_ini,m,pM,pF,b,pl,iM,time)

  1. tmp_ini = variável usada para contar o tempo
  2. m = mapa ([string] com apenas os blocos existentes no mapa)
  3. pM = lista de powerMais
  4. pF = lista de powerFlame
  5. b = lista de bombas
  6. pl = lista de players
  7. iM = lista dos bmp usados para fazer os desenhos
  8. time = contador com o tempo

Esta função permite transformar os dados recebidos num tipo de dados interno para facilitar o desenvolvimento. 

>>> [String] ---> Estado

desenhaEstado :: Estado -> Picture

Função que desenha o jogo.

A razão pela qual temos uma condição, é porque caso os players morram todos, o jogo acabará, apresentando umas imagens a festejar a vitória.

auxiliar_desenho :: [(Int, Int)] -> [(Int, Int, Int)] -> [(Int, Int)] -> [(Int, Int)] -> [(Int, Int)] -> [(Int, Int)] -> [(Int, Int)] -> [Picture] -> Picture

Auxiliar da função desenhaEstado, onde se desenha cada parte independente no ecrã.

  1. p = desenha as paredes
  2. pl = desenha os players
  3. tij = desenha os tijolos (blocos que são removíveis)
  4. bb = desenha as bombas
  5. pM = desenha os powerMais (aumenta a capacidade de bombas)
  6. expl = desenha as explosões
  7. back = desenha o background
  8. pF = desenha os powerFlames (aumenta o raio das explosões)

desenha_players :: [(Int, Int, Int)] -> [Picture] -> [Picture]

Função que desenha os players. Tem de se ter uma diferente da outra desenha, porque os players estão no mapa com um tipo de dados incompativeis com os outros.

desenha :: [(Int, Int)] -> Picture -> [Picture]

Desenha as imagens no ecra, recebendo um par de ordenadas (correspondes a um eixo imaginário no mapa)
